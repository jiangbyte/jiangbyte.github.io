## 分支操作

确保本地分支是最新的：

```bash
git checkout xxx分支
git pull origin xxx分支
```

切分支：

```bash
git checkout xxx分支
```

把当前分支合并到目标分支（先切到目标分支再 merge）：

```bash
git checkout 目标分支
git merge 来源分支
```

创建并切换新分支：

```bash
git checkout -b xxx分支
```

推送新分支到远程：

```bash
git push -u origin xxx分支
```

## 状态保存（git stash）

手头活干到一半，突然要切分支修 bug，又不想提交半成品代码——`git stash` 就是干这个的。

保存当前修改：

```bash
git stash push -m "保存我的当前工作"
```

切出去干完活，回来恢复：

```bash
git checkout 原分支名
git stash pop
```

查看 stash 列表：

```bash
git stash list
```

恢复指定的 stash（注意 Windows 下 `stash@{n}` 要加引号）：

```bash
git stash apply stash@{n}
# Windows
git stash apply "stash@{0}"
```

`git stash` 不产生提交记录，比临时 commit 干净很多。
